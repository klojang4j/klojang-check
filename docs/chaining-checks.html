KLOJANG CHECK

Chaining Checks

As mentioned and illustrated, Klojang Check lets you chain muiltiple checks on a single value. The is() and isNot() methods together constitute a fluent API.

You can also chain checks on different values. This can make for very concise argument validation:

  public void processMessages(List<String> messages, int from, int to) {
    Check.notNull(messages).and(from).is(gte(), 0)
        .and(to).is(gte(), from).is(lte(), messages.size());
    // do stuff ...
  }

Note that with the and() method you are in fact breaking out of the fluent API as it returns a new IntCheck or ObjectCheck instance. The example hopped from an ObjectCheck to an IntCheck to another IntCheck.
