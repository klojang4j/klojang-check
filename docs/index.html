KLOJANG CHECK

Introduction

Klojang Check is a Java module dedicated to defensive programming. It provides a set of syntactical constructs that let you define checks on preconditions and postconditions in a concise and elegant manner. In addition, it comes with a well-curated set of predefined common checks. These checks are associated with short, informative error messages, so you don't have to invent them yourselves.

To use Klojang Check, add the following dependency to your POM file:

<dependency>
  <groupId>org.klojang.check</groupId>
  <artifactId>klojang-check</artifactId>
  <version>1.0.2</version>
</dependency>

The Javadocs for Klojang Check can be found here.

Klojang Check takes a different approach to pre- and postcondition verification than, for example, Guava's Preconditions class and Apache's Validate class. Here is an example of Klojang Check in action:

public class InteriorDesigner {

  private final int numChairs;

  public InteriorDesigner(int numChairs) {
    this.numChairs = Check.that(numChairs).is(gte(), 0).is(lte(), 4).is(even()).ok();
  }

  public void applyColors(List<Color> colors) {
    Check.that(colors).is(notEmpty().and(contains(), noneOf(), RED, BLUE, PINK);
    // apply the colors
  }

  public void addCouch(Couch couch) {
    Check.notNull(couch).isNot(c -> c.isExpensive(), "couch too expensive");
    // add the couch
  }

}
