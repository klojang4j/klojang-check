KLOJANG CHECK

Validating Argument Properties

If the value to be validated is an object, its validity may to a large extent depend on the validity of its properties. Therefore, Klojang Check enables you to check argument properties as part of validating the argument. This is done via the has() and notHas() methods on IntCheck and ObjectCheck. (IntCheck? Yes, see below.) Here is an example:

public class QueryExecutor {

  private final Query query;

  public QueryExecutor(Query query) {
    this.query = Check.that(query, "query")
        .has(Query::where, "where", notEmpty())
        .notHas(Query::offset, "offset", negative())
        .has(Query::limit, "limit", gte(), 10)
        .has(Query::limit, "limit", lte(), 10000)
        .ok();
  }

}

If, in this example, a query would be passed with a limit of 5000000, this would result in an IllegalArgumentException with the message:

query.limit must be <= 10000 (was 5000000)

Common Properties

The CommonProperties class provides some commonly used properties of well-known classes and interfaces — for example the size property of a Collection. As with the CommonChecks class, these properties are already associated with a descriptive name of the property they expose. Thus, very little input from you is required to generate a full-blown, informative error message:

import static org.klojang.check.CommonChecks.gte;
import static org.klojang.check.CommonProperties.size


// stuff ...


    Check.notNull(emps, "employees").has(size(), gte(), 100);

This would cause the following error message to be generated if the size of the emps collection was, say, 42:

employees.size() must be >= 100 (was 42)

Note that the term "property" is somewhat misleading. This first argument to the has() and notHas() methods simply is a function that takes the value being validated and produces some other value, which is then also validated. Thus, the CommonProperties class also contains, for example, an abs() function, which returns the absolute value of an int value. In other words, int values can also have "properties" — like "its" absolute value, or "its" cosine. And hence, the IntCheck class has has() and notHas() methods just like the ObjectCheck class.
